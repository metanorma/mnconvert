ISO Standards Tag Set (ISOSTS) version 1.1: A customization of NISO JATS version 0.4

International Organization for Standardization (ISO)

Change history

Version 1.1, 2013-07-01

Changes since version 1.0:

  • Rename element <cen-meta> to <reg-meta>
  • Delete element <leading-org> and remove it from the <reg-meta> content model
  • Create element <release-version-id> and add it to the <reg-meta> content model
  • Provide content model for element <nat-meta>
  • Allow <email>, <ext-link>, <uri>, <inline-graphic>, <private-char> in TBX highlight elements
  • Allow <break> inside <p>

Version 1.0, 2012-04-26

The only changes are to the version number and release date. The version number has been changed to 1.0 to indicate that this is the first stable release.

Version 0.7, 2012-02-14

Changes since version 0.6:

  • Make <permissions> optional, repeatable in the <iso-meta> and <cen-meta> elements.
  • Added links and citations (<element-citation>, <mixed-citation>, <std>, <fn>, <target>, <tbx:entailedTerm>, and <xref>) to <preformat>.
  • Allow <fig> inside <td>.
  • Allow <p> inside <td> to avoid using <break> for separating Paragraphs.
  • Allow <std> and <std-ref> inside all emphasis elements (<bold>, <italic>, <monospace>, <num>, <overline>, <sans-serif>, <sc>, <strike>, <underline>).
  • Allow simple links (<fn>, <target>, <tbx:entailedTerm>, <xref>) inside <label>.
  • Allow <non-normative-note> after <ref-list> inside <sec>.

Version 0.6, 2011-12-16

Changes since version 0.5:

  • Add change history to the tag library.
  • Make <annex-type> optional.
  • Make <title> optional in the <sec> and <app> elements (but require that at least one of <title> and <label> appear).
  • Allow <break> and <xref> to appear within <mixed-citation> and <element-citation>.
  • Allow <fn> and <fn-group> to appear within <fig>.
  • Allow <non-normative-note> to appear within <ref>, <ref-list>, and <td>.
  • Make <tbx:entailedTerm> a member of %simple-link.class;, so that it is allowed to appear within <p>, since it is sometimes needed in lists and tables occurring within a <tbx:termEntry>. (Note: because the class is used in other places as well, <tbx:entailedTerm> will be syntactically legal in many places; it is semantically appropriate, however, only within term entries.)
  • Allow the @style-type attribute to appear on <p> elements to allow manual specification of styling.
  • Define <sub-part> to handle cases of numbered sub-parts appearing within a part of a standard.
  • Allow <year> to occur within <std-ref>.
  • Change the type of the @position attribute on the <fig> element to CDATA, eliminating earlier constrained list of values.
  • Allow <permissions> to occur within <iso-meta> and <cen-meta>.

Version 0.5, 2011-10-06

First draft version.
